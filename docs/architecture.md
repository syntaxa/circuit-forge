
### **Архитектурный документ фитнес-приложения**

#### **1. Введение: Концепция и цели проекта**

"Circuit Forge" — это прогрессивное веб-приложение (PWA), созданное для проведения эффективных круговых тренировок высокой интенсивности (HIIT) в домашних условиях. Основная миссия проекта — сделать фитнес доступным для всех, независимо от уровня подготовки, наличия времени или доступа к спортивному инвентарю.

**Ключевая концепция** строится на трех принципах:

1.  **Простота и доступность:** Пользователю не нужно специальное оборудование — достаточно собственного веса или обычного стула. Интерфейс интуитивно понятен и не перегружен лишними функциями, позволяя сосредоточиться на тренировке.
2.  **Интеллектуальная персонализация:** В основе приложения лежит умный алгоритм генерации тренировок. Он не просто подбирает случайные упражнения, а учитывает историю занятий пользователя, обеспечивая сбалансированную нагрузку на все группы мышц и внося разнообразие в программу. Это помогает поддерживать мотивацию и способствует гармоничному физическому развитию.
3.  **Гибкость и кастомизация:** Пользователь может настраивать ключевые параметры тренировки (длительность, количество упражнений и циклов), а также редактировать и добавлять собственные упражнения в базу, адаптируя приложение под свои личные цели и предпочтения.

Начальная фаза проекта в виде PWA выбрана стратегически, чтобы быстро запустить продукт, охватить максимальную аудиторию на всех платформах (десктоп, мобильные устройства) и собрать обратную связь с минимальными затратами на разработку.

#### **4. Архитектура приложения (Фаза 1)**

##### **4.0. Подход к формированию программы тренировки**

Описание того, как приложение формирует программу тренировки (реализация — в `workoutGenerator.ts`, см. п. 4.2).

1. **Ротация групп мышц по истории (2 из прошлой + 1 новая)**  
   На каждую тренировку выбирается ровно 3 группы мышц из шести (ноги, пресс, спина, грудь, руки, плечи). При первой тренировке — 3 случайные группы. При последующих — 2 группы из предыдущей тренировки + 1 новая (из оставшихся). Так обеспечиваются преемственность нагрузки, разнообразие и со временем равномерная проработка всех групп мышц.

2. **Ограничение по нагрузке: нет двух «высоких» подряд**  
   В плейлисте не допускается два упражнения с уровнем нагрузки «высокий» подряд. Генератор перетасовывает список (ограниченное число попыток) до соблюдения этого правила, чтобы снизить риск перегрузки и дать передышку между самыми тяжёлыми упражнениями.

3. **Гарантия покрытия выбранных групп**  
   Для каждой из трёх выбранных групп мышц в плейлист гарантированно попадает минимум одно упражнение. Остальные слоты заполняются случайными упражнениями из этих же групп. Ни одна выбранная группа не остаётся без нагрузки.

4. **Генерация «здесь и сейчас», без выбора программы**  
   Пользователь не выбирает готовую программу из каталога. При открытии экрана настройки тренировки план генерируется автоматически: группы мышц по истории, затем плейлист по правилам выше. При необходимости можно перегенерировать только плейлист (кнопка «Обновить план») для тех же групп — получится другой набор упражнений.

##### **4.1. Модель данных (Хранение в `localStorage`)**

*   **`Exercise` (Упражнение):** Группы мышц и сложность заданы enum-ами `MuscleGroup` и `Difficulty` (значения: 'ноги' | 'пресс' | 'спина' | 'грудь' | 'руки' | 'плечи' и 'лёгкий' | 'средний' | 'высокий'). Поле `source` различает встроенные упражнения и пользовательские.
    ```typescript
    type ExerciseSource = 'base' | 'user';

    interface Exercise {
      id: string;
      name: string;
      description: string;
      muscleGroup: MuscleGroup;
      difficulty: Difficulty;
      biSided?: boolean;  // выполнение на обе стороны
      steps: string;      // пошаговое описание
      source?: ExerciseSource;  // 'base' — из SEED_EXERCISES, 'user' — свои
    }
    ```
*   **Ключи хранилища:** `cf_user_exercises` (пользовательские упражнения), `cf_deactivated_base_ids` (идентификаторы отключённых упражнений из базы), `cf_hide_clone_warning` (не показывать предупреждение при клонировании), `cf_settings`, `cf_profile`, `cf_history`. Раньше использовался `cf_exercises` — при первом обращении `runMigration()` удаляет его и инициализирует `cf_user_exercises` пустым массивом.
*   **`Settings` (Настройки):**
    ```typescript
    interface Settings {
      exerciseDuration: number;   // секунды, дефолт 30
      exercisesPerCycle: number; // дефолт 10, минимум 9
      cycleCount: number;         // дефолт 2
      ttsVoiceURI: string | null;
    }
    ```
*   **`UserProfile` (Профиль пользователя):** Тип определён в `types.ts`; ключ `cf_profile` в хранилище предусмотрен, персистенция профиля в текущей реализации может не использоваться.
    ```typescript
    interface UserProfile {
      userId: string;
      userName: string;
    }
    ```
*   **История тренировок (`WorkoutLog`):**
    ```typescript
    interface WorkoutLog {
      date: string;                 // ISO 8601
      muscleGroupsUsed: MuscleGroup[]; // массив из трёх групп мышц
    }
    ```

##### **4.2. Сервисы и логика (Бизнес-логика)**

*   **`storageService.ts`:** Единый сервис работы с `localStorage`.
    *   **Миграция:** `runMigration()` — при наличии старого ключа `cf_exercises` и отсутствии `cf_user_exercises` удаляет старый ключ и инициализирует `cf_user_exercises` пустым массивом.
    *   **Упражнения:** «Стартовый набор» хранится в коде (`constants.ts` → `SEED_EXERCISES`) с `source: 'base'`. Пользовательские упражнения — в `cf_user_exercises` с `source: 'user'`. `getExercises(forWorkout?)` возвращает объединённый список (база + пользовательские); при `forWorkout === true` исключает упражнения, чьи id есть в `cf_deactivated_base_ids`. Отдельно: `getExercisesForWorkout()`, `getUserExercises()`, `saveUserExercises()`.
    *   **Отключение упражнений из базы:** `getDeactivatedBaseIds()`, `setDeactivatedBaseIds()`, `toggleBaseActive(id)`, `isBaseDeactivated(id)` — управление списком id встроенных упражнений, которые пользователь отключил (например, после клонирования для редактирования).
    *   **Настройка предупреждения о клонировании:** `getHideCloneWarning()`, `setHideCloneWarning(value)`.
    *   **Прочее:** CRUD для `Settings`; история тренировок (`getHistory`, `addLog`, `getLastLog`, `clearLastLog`, `clearHistory`); ключ профиля `cf_profile` предусмотрен.
*   **`workoutGenerator.ts`:** Реализует подход к формированию программы тренировки (см. разд. **4.0**). Использует `StorageService.getExercisesForWorkout()` (только активные упражнения: база без отключённых + пользовательские). Методы: `selectMuscleGroups()` — выбор 3 групп мышц по истории (последний лог); `generatePlaylist(targetMuscles, count)` — гарантия минимум одного упражнения на каждую выбранную группу, заполнение слотов с приоритетом неповторяющихся упражнений, затем ограничение «нет двух упражнений с высокой нагрузкой подряд» (через разделение на HARD и не-HARD и чередование или перетасовку), и устранение подряд идущих одинаковых упражнений (`avoidConsecutiveDuplicates`). Порядок упражнений во всех циклах идентичен (один и тот же плейлист повторяется).
*   **Логика активной тренировки** реализована в компоненте **`ScreenWorkout`** (отдельного `WorkoutEngine` нет): состояния (подготовка PREP, работа WORK, пауза PAUSED, завершена FINISHED), таймер упражнения, пауза/продолжение, отмена с подтверждением, автоматические переходы между упражнениями и циклами.
*   **`ttsService.ts`:** Абстракция над браузерным Web Speech API, управляет озвучкой и выбором голосов.
    *   **Звуковые сигналы и анонсы:**
        *   Анонс следующего упражнения перед его началом (название упражнения).
        *   Обратный счёт "3, 2, 1" перед началом каждого упражнения.
        *   Обратный счёт "3, 2, 1" перед окончанием упражнения (за 3 секунды до завершения).
        *   Анонс смены стороны в середине упражнения (для упражнений, требующих выполнения на обе стороны, если применимо).
        *   Озвучка завершения упражнения и перехода к следующему.

##### **4.3. Навигация и состояние приложения**

*   **Экраны (`AppScreen`):** `SETUP`, `WORKOUT`, `SETTINGS`, `DATABASE`, `DATABASE_ABOUT`, `ABOUT`.
*   **Маршрутизация:** В `App.tsx` состояние `currentScreen` и функции навигации; детальный просмотр упражнения вынесен в портал (`ScreenExerciseDetail` поверх любого экрана при `exerciseDetail !== null`).
*   **Актуальность плейлиста:** При возврате на экран настройки тренировки (`SETUP`) увеличивается `setupRefreshKey`, за счёт чего `ScreenSetup` перемонтируется и плейлист заново строится из актуальной базы (важно после редактирования упражнений на экране «База упражнений»).

##### **4.4. Структура компонентов (UI)**

Все компоненты интерфейса должны быть адаптированы для работы на экранах смартфонов с учетом ограничений мобильных устройств (размер экрана, touch-интерфейс, ориентация экрана).

*   **`ScreenSetup`:** Главный экран с кнопкой старта и отображением выбранных на сегодня групп мышц; кнопки перехода в настройки, базу упражнений и «О приложении». Кнопки «Обновить план» и «Забыть историю и обновить» для перегенерации плана.
*   **`ScreenSettings`:** Форма настроек, включая выпадающий список для выбора голоса TTS.
*   **`ScreenDatabase`:** Интерфейс для просмотра, добавления, редактирования и удаления упражнений. Фильтры по источнику («Все», «База», «Моё»), по названию и группе мышц; счётчики по группам. Упражнения из базы нельзя редактировать напрямую: при нажатии «Редактировать» показывается предупреждение о клонировании (можно отключить в настройках экрана), затем создаётся копия в «Моё» и оригинал отключается в подборке. Кнопка перехода на экран «О базе упражнений».
*   **`ScreenDatabaseAbout`:** Экран с текстовым описанием работы базы упражнений (фильтры, клонирование, хранение на устройстве).
*   **`ScreenWorkout`:** Экран активной тренировки с таймером и управлением.
    *   **Визуальный индикатор времени:** Кольцевой прогресс-бар, который закрашивается по мере выполнения упражнения, показывая оставшееся время.
    *   **Отображение текущего упражнения:** Название упражнения, описание, группа мышц, уровень нагрузки.
    *   **Индикация цикла:** Отображение текущего цикла и общего количества циклов.
    *   **Индикация прогресса:** Отображение текущего упражнения в цикле и общего количества упражнений.
    *   **Элементы управления:**
        *   Кнопка "Пауза" (отображается во время активной тренировки).
        *   Кнопка "Продолжить" (отображается во время паузы).
        *   Кнопка "Отменить тренировку" (с подтверждением действия).
    *   **Адаптивность:** Все элементы интерфейса оптимизированы для touch-интерфейса и различных размеров экранов.
*   **`ScreenExerciseDetail`:** Экран/оверлей с детальной информацией об упражнении (описание, шаги выполнения). Открывается из экрана настройки тренировки и из экрана тренировки.
*   **`ScreenAbout`:** Экран «О приложении».

#### **Публикация (GitHub Pages)**

При каждом пуше в ветку `main` запускается GitHub Actions workflow (`.github/workflows/deploy-pages.yml`): выполняется сборка production-приложения (`npm run build`) и публикация артефакта для GitHub Pages.

**Чтобы сайт открывался по адресу репозитория:**

1. В репозитории: **Settings → Pages**.
2. В разделе **Build and deployment** выберите **Source: GitHub Actions**.
3. После первого успешного деплоя сайт будет доступен по адресу вида `https://<owner>.github.io/<repo-name>/`.
