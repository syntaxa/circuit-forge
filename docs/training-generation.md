# Генерация программы тренировки

Этот документ описывает актуальный алгоритм составления плейлиста упражнений для одной тренировки.

## Цели алгоритма

- Сохранять ротацию групп мышц между тренировками по истории пользователя.
- Гарантировать, что каждая выбранная на тренировку группа мышц представлена хотя бы одним упражнением.
- По возможности не ставить подряд упражнения одной и той же группы мышц.
- По возможности не ставить подряд два тяжелых упражнения (`HARD`), но это правило вторично.
- По возможности не ставить подряд два одинаковых упражнения.

## Выбор групп мышц

Сейчас генератор по-прежнему выбирает 3 группы мышц:

- Если истории нет, берутся 3 случайные уникальные группы.
- Если история есть, берутся 2 группы из предыдущей тренировки и 1 новая группа из оставшихся.

Это правило определяет состав тренировки, но не порядок упражнений внутри нее.

## Формирование пула упражнений

После выбора целевых групп генератор:

1. Берет только активные упражнения из выбранных групп мышц.
2. Сначала старается использовать уникальные упражнения по `id`.
3. Гарантирует минимум одно упражнение на каждую выбранную группу мышц.
4. Если уникальных упражнений не хватает, добирает плейлист повторами.

## Приоритеты порядка упражнений

Финальный порядок строится не случайной перетасовкой, а отдельным шагом упорядочивания.

Приоритеты такие:

1. Не ставить подряд одинаковый `id`, если доступна альтернатива.
2. Не ставить подряд одну и ту же `muscleGroup`, если доступна альтернатива.
3. Избегать связки `HARD -> HARD`, если это не ломает более важные ограничения.

Иными словами, чередование групп мышц важнее, чем разделение тяжелых упражнений.

## Fallback-поведение

Иногда все ограничения одновременно выполнить невозможно:

- упражнений мало;
- почти все упражнения относятся к одной группе;
- в пуле слишком много повторов;
- почти все доступные упражнения имеют сложность `HARD`.

В этом случае генератор не должен возвращать пустой или нестабильный результат. Он выбирает порядок с минимально возможным числом нарушений:

- сначала старается не повторять `id`;
- затем старается разнести группы мышц;
- затем, если возможно, разносит `HARD`.

## Что изменилось в версии 1.5.0

Раньше основным ограничением было отсутствие двух `HARD` подряд, а порядок по группам мышц мог получаться неудобным.

Начиная с `1.5.0`:

- основной принцип порядка — перемежение групп мышц;
- ограничение на два тяжелых упражнения подряд стало мягким;
- тесты генератора проверяют именно этот приоритет.
